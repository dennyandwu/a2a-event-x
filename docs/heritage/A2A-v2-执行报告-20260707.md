## 二、Incident:EBADF-20260706(issac 消费停滞)

- **现象**:07-06 16:21 起 issac 对 sop-email-check 停止 ack,积压 18 条至 20:39。
- **根因**:exec spawn EBADF——session 收到 Hook-C 唤醒但执行层无法 spawn 子进程,跑不动 a2a CLI。证据:gateway.log 共 294 处 EBADF;07-04 05:05、07-05 15:59(×2)gateway 因 "exec spawn EBADF persistent" 被请求重启;07-06 当天 a2a-log.py 运行失败 5 次(07:45/10:03/14:01/16:07/16:14),末两次发生在 issac 的 discord session 内,紧邻积压起点;Hook-C wake_sent 全程正常。
- **性质**:FD 泄漏家族。session-fd-relief 归档(现每日两跑)只治目录膨胀,不释放已打开 FD;**巡检时 gateway FD 已达 8,871(危险阈值 9,000)**。
- **处置**:残余积压已清(P0-7a);watchdog 已加 FD 阈值告警(warn 6000 / crit 9000)。
